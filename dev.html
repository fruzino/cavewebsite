Developer Hub

Cave is an open ecosystem designed for extensibility. This guide explains how to build and integrate plugins into the Cave interpreter.

Plugin Architecture

The Cave interpreter interacts with plugins using a specialized RPC-like protocol over standard input/output. This allows you to write plugins in any language.

Directory Structure

Plugins must be placed in the c:\Cave\plugins\ directory with the following structure: 

c:\Cave\plugins\
└── my_plugin\
    ├── run.bat      <-- Entry point
    └── main.py      <-- Logic (optional)

Communication Protocol

When you use #insert my_plugin in a Cave script, the interpreter executes the run.bat file and communicates via the following protocol:

Direction Format Description
Cave -> Plugin CALL <method> <args> Sent to plugin's `stdin` when a method is called.
Plugin -> Cave RESULT <value> Optional: Print to `stdout` to return data to Cave.

CLI Commands

The Cave CLI provides tools for managing your environment and community plugins.

Command Description
cave cli install <name> Install a plugin from the repository.
cave cli update <name> Update an installed plugin to the latest version.
cave cli remove <name> Remove a plugin from the system.

Contributing

Cave is driven by the community. To contribute a plugin, add it to the plugins/ folder in our repository.

$ git clone https://github.com/fruzino/cavelib
$ cd cavelib/plugins
$ # Create your plugin folder here
